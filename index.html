Mission (Im)Possible: Discovering Autopilot Logic Flaws in Open-Source Drone Frameworks with LLM Agents

Abstract

Drone systems are increasingly deployed across a wide range of domains, including environmental monitoring, infrastructure inspection, agriculture, etc., due to their automation capabilities. However, these systems can harbor subtle logic vulnerabilities that may lead to significant security risks. With recent advances in large language models (LLMs), we observe that, when equipped with essential domain knowledge, LLMs demonstrate a strong capability to understand the relationships between variables and functions, making them well-suited for reasoning about complex mission control logic in drones.

We introduce Impossible Mission Finder} (IMF), a fully automated framework combining both static code analysis and dynamic execution with LLM agents, to find mission-planning logic flaws in open-source drone frameworks. IMF initiates analysis by performing code slicing on navigation-related modules of open-source drone frameworks, to extract function-level slicing related to mission-critical operations. An LLM agent then analyzes these slices to reason about affected variables, control conditions, and inter-module dependencies and generates logically consistent test scenarios that are further translated into executable test cases. Finally, such test cases will be executed and an LLM-driven log analyzer will dynamically validate whether vulnerabilities occur. We evaluated IMF on PX4, which is a major open-source drone framework, and IMF successfully identified 12 previously unknown logic vulnerabilities.
